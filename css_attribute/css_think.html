1.Html：事先根据所有的页面划分公用或重复模块，再按模块唯一性分配给每个人，有人负责搭建框架，有人制作模块，最后合并框架和模块。

2.Html：注意文档内容的顺序，不要去看效果图的视觉布局，要考虑文档内容重要性为起点，按有意义的顺序来安排html文档流的顺序。

3.Html：网页的布局是种楼层的概念，一层一层的搭建，就像我看的漫画一样，是横向看的，然后再往下看，也可以这样理解，大多数页面都是向下滚的，所以我们就一行一行布局，如果页面是横向滚的，那我们就列列布局。

4.Html：内容的css不参与外部与其他外部便签之间的布局，也就是不跨标签布局，内容和框架的分离。

5.Html：什么时候用组合，什么时候用继承。（模块与模块之间的公用性-继承-基类，他的不一样的地方，扩展性-组合-扩展类，模块与周围模块之间的接口-组合）

6.Css：layout.css-style.css。（从css的各自属性用途来分）

7.Css：base.css-global.css-page.css-js.css。（从整站的架构来分）

8.Css：css的选择器临近写，不越级写，需要提高权重的时候，一级级往外加。

9.Css：第一阶段是在一个css文件中把多个页面按自己的习惯顺序从上往下编写样式，基本不考虑有无公用样式，以完成设计呈现为首要目的；第二阶段是提取不同页面 中的通用样式，如公用颜色、图标、按钮等，实现一些基本元素的复用；第三阶段是提取公用功能模块，如导航、版权信息等，实现部分公用模块的复用。

10.面向对象的CSS有两个原则，独立的结构和样式，独立的容器和内容。独立的结构和样式就是把布局样式和设计样式独立出来。独立的容器和内容所指的是把内容从容器中分离出来，换过句话说任何对象（容器），应该适应接受任何形式的内容。

11.以10为单位不同的宽度和间距。

12.有个问题:就是css模块和不是模块的命名，如果一个东西一会是模块一会不是模块怎么命名，如果一个大模块里面的子内容，搬到其他地方自己变一个独立的模块了，这些要怎么命名

13.还是根据他是什么来命名，比如按钮就是按钮他不会变成其他的，而如果命名模块的话，他一会儿是模块一会儿不是模块很麻烦

14.内容样式和布局样式分离，用下来发现嵌套太深了，现在就是想不用便签来区分内容和布局，而是在公用的标签上面利用class来区别，用组合的方式，一个是控制内容的class，剩下就留给布局的class了，布局不限class

15.现在我这样分组件和模块的区别，比如，按钮是组建，标题是组件，也就是一个标签搞定的样式，模块用标题来区分，一个标题包含的所有内容都看做模块，里面有文章图片按钮等等。

16.模块还是需要统一加class，来统一获取到里面的元素加样式，不要为了图省事而不再模块上统一价class，而用里面组件的class通过兄弟选择器来加样式，如果里面要换个其他组件的话，样式就乱了，这样写是非常脆弱的。
