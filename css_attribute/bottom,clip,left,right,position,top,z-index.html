绝对定位也能作为绝对定位的包含块，不过包含块是fixed还是relative，里面的绝对定位的宽度在不显性设置的情况下的宽度不超过他包含快的宽度。不过当带有数字或是字母同时没有空格的情况下，宽度就被这些硬撑开来了。

包含块绝对定位第一层绝对定位第二层，这层的绝对定位并不是按照祖先元素的相对定位来的确定，她的top，right，bottom，left是跟着最近的定位来的

定top和bottom两个点，元素有高度了，定left和right两个点，元素有宽度了，而再在绝对定位元素上显性的设置高和宽后，top，right，bottom，left就像这个绝对定位元素外面又隐性的多了一个包含块，居然就可以是定位元素居中了

包含块居中元素

定位元素还能垂直居中，正常文档流就不行

在不设z-index的情况下，定位的层级关系还是按照文档流的顺序来的，越在后面越在上面。

绝对定位元素有设top,right,bottom,left四个值时，设有高度的情况下bottom无效，设有宽度的情况下right无效。

4个值的时候，再加正外边距内边距

当绝对定位元素没有文本的情况下，4个值需全设才有效果，因为absolute有包裹性，有文本设1个就有效果

4个值的时候，再加负外边距，有增加宽高的效果，而在正常文档流中只有增加水平宽度的效果，没有垂直效果，哪边负值需要增加效果，那哪边就要设定位置，否则的情况就像在正常文档流中的一样，为什么？还是因为absolute的包裹性在起作用

负值的时候

当水平或垂直的2个相对值之间为0或者是负的话，那绝对定位元素就消失了，留下的文本内容还是以top和left的位置来确定

clip只应用于绝对定位

可以用px，em，但不能用%。

定位left跟着定位元素的左边框，right跟着元素的右边框，top跟着元素的上边框，bottom跟着元素的下边框。

单设fixed，或是设right,left，不设top,bottom的值，元素就会消失，他不像absolute一样会根据文档流的位置，停留在相对的原地。

这张图说明了为什么在fixed的情况下只设left，right是没有效果的，这是张firebug的图，可以看到纵向的默认位置是认可文档高度的，而横向的距离就只视窗口位置来，不管body或是html设了多长的宽度都没用，也就是说在不设4个值，单设fixed的情况下，left和right已经符合作为控制fixed属性的条件了，之所以元素还没有出来，是因为垂直的属性还没有符合条件，还不能确定她在平面当中的位置，所以不能显示。
